# Glossary

- **Application session** — a separate, expiring server authorization created after a successful ceremony. Usually represented by a cookie or bearer token.
- **Attestation** — evidence about an authenticator and the provenance of a newly created credential. Consumer services often request `none` to minimize privacy and trust complexity.
- **Authenticator** — hardware or software that stores credential private keys and performs credential creation and signing after user consent.
- **Authenticator data** — signed binary structure containing RP ID hash, flags, signature counter, and optional credential/extension data.
- **Backup eligibility (BE)** — signed flag indicating that a credential can be backed up, including multi-device/synced Passkeys.
- **Backup state (BS)** — signed flag indicating that a backup-eligible credential is currently backed up.
- **CBOR** — Concise Binary Object Representation, used for attestation objects and COSE keys.
- **Ceremony** — the coordinated registration or authentication interaction between user, client, authenticator, and RP.
- **Challenge** — a fresh, unpredictable, single-use server nonce that prevents replay.
- **Client data** — JSON bytes containing ceremony type, challenge, origin, and cross-origin context.
- **COSE** — standards for representing cryptographic keys and algorithms in CBOR. WebAuthn credential public keys are COSE_Key values.
- **Credential ID** — opaque authenticator-generated bytes that select a credential/public key.
- **Discoverable credential** — a credential the authenticator can find from RP ID without an allow-list. Passkeys use this account-selection model.
- **ES256** — ECDSA over the P-256 curve with SHA-256, represented by COSE algorithm `-7`.
- **Origin** — scheme, host, and port of a client context, such as `https://passkeys.example.com`.
- **Passkey** — a user-focused deployment of a phishing-resistant WebAuthn public-key credential, including platform UX and credential lifecycle.
- **Relying party (RP)** — service that registers and authenticates users with WebAuthn credentials.
- **RP ID** — domain-like scope of a credential. Authenticator data contains its SHA-256 hash.
- **Signature counter (`signCount`)** — authenticator value that may help detect duplicated credential use; synced Passkeys may keep it at zero.
- **User handle** — stable, opaque RP-generated bytes stored with a discoverable credential to identify the RP-local account.
- **User presence (UP)** — signed result showing that the user performed an authorization gesture.
- **User verification (UV)** — signed result showing that the authenticator locally verified the user through an accepted method.
- **WebAuthn** — W3C API, data model, and relying-party/authenticator verification framework for public-key credentials.
